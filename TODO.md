# FORGE Roadmap

## Priority 1: Production Trust
- Add tests for cloud sync: offline create, reconnect, replay, and remote snapshot refresh.
- Replace native AsyncStorage storage with platform secure storage before real field use.
- Add a privacy screen explaining local storage, cloud sync, exports, wipe, and GPS route handling.

## Priority 2: First-Use Clarity
- Keep first-run choice: Start Fresh or Load Demo Data.
- Default new users to empty personal data.
- Keep demo data available only as an explicit mode.

## Priority 3: Readiness Engine
- Keep merging readiness check-in signals with recent training load.
- Add clearer reason codes for Push, Maintain, Reduce, and Recover.
- Add integration tests proving readiness changes Home recommendations.

## Priority 4: Ruck Reliability
- Add a post-ruck review screen before saving.
- Store route confidence, rejected point count, GPS accuracy, and paused time.
- Add a simpler Field Mode that hides planning tools during active tracking.

## Priority 5: Device QA
- Test installed PWA on iPhone Safari and Android Chrome.
- Test GPS permission denied, weak GPS, offline shell, reload during active ruck, and export/import restore.
- Test bottom tabs, keyboard inputs, PIN lock, and OPSEC wipe on real phones.
